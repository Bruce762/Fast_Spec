# 目的

對於使用者針對已存在的 Bug 修復方案提出的改動或補充，先調研並查閱 `@fastplan/bug/[提案簡稱]_bug_revise_plan.md` 與相關程式資料，完成問題拆解後，將修訂內容更新至 `@fastplan/bug/[提案簡稱]_bug_revise_plan.md`（不需實作）。

# 作業流程

1. 針對使用者提出的改動或補充需求進行初步調研。
2. 查閱 `@fastplan/bug/[提案簡稱]_bug_revise_plan.md`。如需了解原始功能脈絡,可參考 `@fastplan/plan/[提案簡稱]_plan.md`(如果存在)。同時查看其他相關程式與資料。
3. 比較與原本方案的差異，以及如果有疑問請向使用者提問，然後綜整調研結果，將問題適度拆解為可執行的子問題。
4. 依拆解結果修訂方案內容，並將修正後的內容更新至 `@fastplan/bug/[提案簡稱]_bug_revise_plan.md`，且不需實作此檔案到程式。
5. 創建 `@fastplan/bug/[提案簡稱]_bug_revise_plan_modify.md`，並把此次修改用簡單易懂少字的方式告訴使用者。

# 命名規範

* **提案簡稱** ：使用小寫、連字號，例：`task-refactor`、`onboarding-flow`。
* **路徑固定** ：`@fastplan/bug/[提案簡稱]_bug_revise_plan_modify.md`。

# 使用語言

使用與使用者相同的語言即可
