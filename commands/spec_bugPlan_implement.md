# 目的

依照 `@fastplan/bug/[提案簡稱]_bug_revise_plan.md` 的內容去實施 Bug 修復程式碼，且實施時需依照里程碑所條列的子清單任務去逐步勾選並且實作。

# 作業流程

* 依照 `@fastplan/bug/[提案簡稱]_bug_revise_plan.md` 的規劃實施 Bug 修復程式碼開發。
* 以「里程碑 → 子任務」為主軸，逐項勾選、逐步完成。
* 修復過程中若發現新的問題或需要調整方案，應記錄並考慮是否需要更新 `_bug_revise_plan.md`。

# 處理新發現的問題

- 若發現新的 Bug 或需求變更：請記錄於對應的 `_bug_revise_plan.md` 的「待解決問題」章節
- 若問題重大：建議暫停實作,使用 `/spec_bugPlan_modify` 更新方案

# 使用語言

使用與使用者相同的語言即可
